# 🎨 Project Theta Style Guide

This document defines the visual identity and tone of Project Theta to ensure consistency across GitHub, YouTube, and other community channels.

---

## 🏛️ Brand Voice & Tone

### **"It's Just AI."**
- **Approachability:** Use plain English. Avoid gatekeeping jargon. If a complex term is used, explain it immediately.
- **Empowerment:** Don't talk down to the audience. Treat them as intellectual equals who simply lack specific context.
- **Radical Openness:** Every resource is free. Every contribution is valued.

---

## 🎨 Color Palette

| Color | Hex | Usage |
|-------|-----|-------|
| **Deep Indigo** | `#1A1B4B` | Primary Backgrounds, Dark Mode UI |
| **Electric Blue** | `#3D5AFE` | Accents, Call-to-Actions, Links |
| **Bright Cyan** | `#00E5FF` | High-lighted terms, Secondary Accents |
| **Neutral White** | `#F5F5F7` | Body Text, High Contrast elements |

---

## 🔠 Typography

- **Headings:** Inter or Roboto (Sans-Serif). Bold, clean, and authoritative.
- **Body:** Inter or system-ui (Sans-Serif). High readability, generous line spacing.

## 📝 Formatting Guidelines (GitHub Markdown)

1. **Use GitHub Alerts:**
   - Use `> [!IMPORTANT]` for critical concepts.
   - Use `> [!TIP]` for practical shortcuts.
2. **Tables for Comparison:**
   - Always use tables when comparing AI models or tools.
3. **Mermaid Diagrams:**
   - Use Mermaid for technical flows (e.g., how a Transformer works).
